# About

## Author focus

My work and learning currently focus on:

- governance, risk, and compliance;
- ISO 27001 implementation and control analysis;
- identity and access management;
- business continuity and operational resilience;
- security documentation for small organisations;
- evidence-based security reviews.

I am particularly interested in situations where formal requirements appear complete on paper but fail in everyday operations.

Examples include:
- approvals that exist only in email;
- registers that contain incomplete or outdated fields;
- controls that have no clear owner;
- security procedures that cannot be verified with evidence.

## Approach

The projects combine:
- structured documentation;
- simulated business scenarios;
- practical workflows;
- control and risk analysis;
- reusable registers and templates;
- implementation and verification evidence.

The objective is to move beyond checkbox compliance and show how a security requirement can become an operational process.

## AustraKey Lab

AustraKey Lab is a fictional small-company environment used across several portfolio projects.

It provides a consistent business context for demonstrating:
- access management;
- control implementation;
- risk treatment;
- evidence collection;
- security governance;
- process improvement.

A shared environment makes it possible to show how separate security activities affect the same organisation over time.

## Scope and limitations

The material in this knowledge base is created for portfolio, educational, analytical, and demonstration purposes.

Some artefacts are based on practical experience, while others are simulations or reusable models. They should be adapted to the legal, technical, organisational, and risk context of a real organisation before operational use.

The documentation does not represent legal advice, certification assurance, or a complete substitute for an organisation-specific security assessment.
